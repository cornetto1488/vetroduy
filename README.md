# ВЕТРОДУЙ

Десктоп-приложение в стиле «жидкое стекло»: комнаты-каналы как в Discord, внутри каждой — встреча Яндекс Телемоста. Плюс музыкальный бот, который заходит в канал и играет музыку всем.

## Как пользоваться

1. В настройках (⚙) один раз впиши **своё имя** — все экраны Телемоста («Продолжить в браузере», имя, «Подключиться») будут прокликиваться автоматически: клик по комнате = сразу в звонке.
2. Слева нажми **«⌂ Телемост»** — создай встречу и скопируй ссылку.
3. Нажми **«+»**, дай комнате имя и вставь ссылку (`https://telemost.yandex.ru/j/...`).
4. Клик по другой комнате отключает от текущей и подключает к новой — как войс-каналы в Discord.
5. У комнаты со звонком показывается число участников (👤 N); когда кто-то заходит — звук + уведомление Windows.

## Музыкальный бот 🎵

Внизу сайдбара: напиши название радио (например «lofi», «phonk», «Европа Плюс») и нажми ▶ — появится список станций, выбери одну. В канал зайдёт участник «🎵 Музыка», и музыку услышат **все в комнате**. Можно вставить и прямую ссылку на аудиопоток (mp3/радио).

Можно искать и **обычные песни** (не только радио) — движок `yt-dlp` вшит в сборку, ищет по YouTube. Пример: «Сукины дети 1.Kla$».

- ⏸ — пауза / продолжить;
- ⏭ — следующий трек в очереди;
- 🦆 — **дакинг**: музыка сама притихает (до 20%), когда кто-то говорит, и возвращается в тишине;
- ⏻ — выгнать бота;
- ползунок — громкость музыки у всех.

Найденные треки/станции кликом добавляются в **очередь** — играют по порядку, автопереход к следующему.

**Общий бот (синхронно у всех в комнате):** если подключён бэкенд, на комнату живёт максимум **один** музыкальный бот. Кто первым включил музыку — тот «хост» (у него реально крутится бот). Остальные не поднимают своего второго бота, а **правят ту же очередь, громкость, паузу, скип** — всё синхронизируется через Firebase. Выгнать бота может **любой**. Если хост вышел из звонка — музыка останавливается (любой может включить заново и стать новым хостом).

**Права хоста (🔒):** у хоста в контролах музыки есть кнопка 🔒 — там можно «Запретить всем» (управляю только я) или заблокировать конкретных участников. У заблокированных кнопки управления музыкой становятся неактивными с подписью «🔒 хост ограничил управление».

**🎧 ИИ-диджей:** кнопка 🎧 рядом с поиском — сам находит и сразу ставит трек. Напиши артиста/песню → включит топ-результат. Оставь пусто или напиши «что-нибудь / чиллого / качовое» → подберёт под настроение и включит случайный трек.

**🎬 Кинотеатр (watch party):** кнопка 🎬 в шапке. Ведущий вставляет ссылку на видео (RuTube, VK Видео или прямой .mp4 — НЕ YouTube, он в РФ заблокирован) и жмёт «Показать всем» — у остальных открывается то же видео и синхронизируется (перемотка, пауза/плей следуют за ведущим). Каждый слышит звук из своей копии. Источник-независимо: рулим тегом `<video>` на странице.

Технически: скрытая вкладка заходит в звонок отдельным участником, а её «микрофон» подменён на аудиопоток (WebAudio → MediaStream). Поток гоняется через локальный прокси, чтобы обойти CORS.

## Чат, «кто где сидит» и пинги 💬

Эти три функции работают через общий бэкенд (Firebase Realtime Database, бесплатно). Адрес бэкенда берётся из того же манифеста — поле `"backend"`.

- **💬 Общий чат** (кнопка в шапке) — сообщения между всеми, у кого приложение; не теряются при выходе из звонка.
- **👥 Presence** — рядом с комнатой видно, сколько человек там сидит, даже если ты туда не заходил.
- **🔔 Пинги** — наведи на комнату, нажми колокольчик: придёт уведомление, когда в пустой комнате кто-то появится.

### Как включить (2 минуты, один раз)

1. console.firebase.google.com → **Создать проект** (можно без Google Analytics).
2. Слева **Build → Realtime Database → Create Database** → выбери регион → запусти в **тестовом режиме** (или в правилах поставь `{"rules":{".read":true,".write":true}}` — для дружеского чата ок).
3. Скопируй URL базы вида `https://ТВОЙ-ПРОЕКТ-default-rtdb.firebaseio.com`.
4. Добавь его в `vetroduy.json` рядом с `rooms`:
   ```json
   "backend": "https://ТВОЙ-ПРОЕКТ-default-rtdb.firebaseio.com",
   ```
5. Закоммить `vetroduy.json` — у всех друзей чат/presence/пинги включатся автоматически (кнопка 💬 появится в шапке).

## Остальные фишки

- **✕** сворачивает в трей (звонок живёт); выйти — правый клик по трею → «Выйти». Из трея можно переключать комнаты.
- **◱** — мини-режим поверх всех окон (для игр).
- **Ctrl+Shift+M** — мут/анмут микрофона глобально.
- **Демонстрация экрана** — своё окно выбора: весь экран (со звуком системы) или отдельное окно.
- **Рескин Телемоста**: внутри приложения скрыта вся яндексовая обвязка (шапка 360, реклама, «скачайте приложение»), тёмный фон и фирменные фиолетовые кнопки — при этом обычная ссылка на встречу продолжает работать у людей в браузере.

## Общий список комнат (как «сервер» для друзей)

⚙ → вставь https-ссылку на JSON:

```json
[
  { "name": "Общий войс", "url": "https://telemost.yandex.ru/j/11111111111111" },
  { "name": "Дота 2", "url": "https://telemost.yandex.ru/j/22222222222222" }
]
```

Комнаты появятся в разделе «Общие» ☁ у всех с той же ссылкой (обновление раз в минуту). Где бесплатно хранить JSON:

- **GitHub Gist** (проще всего): gist.github.com → файл `rooms.json` → кнопка Raw → эту ссылку в приложение. Задержка обновления до ~5 минут.
- **Firebase Realtime Database** (почти мгновенно): console.firebase.google.com → Realtime Database → правила `".read": true` → ссылка вида `https://проект-default-rtdb.firebaseio.com/rooms.json`.
- **npoint.io** — онлайн-редактор JSON с публичной ссылкой.

## Запуск и сборка

```powershell
npm install
npm start        # запуск из исходников
npm run dist     # установщик: dist\VETRODUY-Setup-1.3.0.exe
```

Установщик — один exe, шлёшь друзьям, установка в один клик.

## Linux (CachyOS / Arch и др.)

Скачай `VETRODUY-Setup-X.Y.Z.tar.gz`, распакуй и запусти:

```bash
tar -xzf VETRODUY-Setup-*.tar.gz
cd VETRODUY-Setup-*
chmod +x vetroduy
./vetroduy
```

Приложение само стартует без песочницы (`--no-sandbox` вшит для Linux), поэтому обычной `./vetroduy` достаточно. Если вдруг всё равно не запускается — запусти явно `./vetroduy --no-sandbox` и, если ругается на права, сначала `chmod +x vetroduy`. На Wayland для демонстрации экрана нужен портал (`xdg-desktop-portal`), в KDE/GNOME обычно уже стоит.

## Технические детали

- Electron; окно без рамки; «жидкое стекло» на CSS; иконка и логотип — build/icon.ico и src/renderer/logo.png.
- Телемост через `<webview>`; одна комната за раз; счётчик участников читается с кнопки «Участники N» в звонке.
- Автовход и мут — программные клики по кнопкам Телемоста (если Яндекс сменит разметку, селекторы в src/renderer/app.js придётся обновить).
- Поиск радио — открытый Radio Browser API (несколько зеркал).
- Данные — `%APPDATA%\vetroduy\config.json` (конфиги старых версий подхватываются автоматически).
